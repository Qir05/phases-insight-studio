\# Claude Project Rules



\## Working Rules

\- Always read this file before making changes.

\- Plan first before editing files.

\- Identify affected files before coding.

\- Make small safe changes only.

\- Do not rewrite the whole project unless asked.

\- Do not remove working features or existing logic.

\- Do not touch `.env` or expose API keys.

\- Keep changes compatible with Vercel.



\## QA Rules

Before saying done:

\- Run `npm run build` if available.

\- Run `npx tsc --noEmit` if available.

\- Check mobile responsiveness.

\- Check desktop layout.

\- Check for console/build errors.

\- Summarize what files changed.



\## UI Rules

\- Make UI clean, polished, and mobile-friendly.

\- Avoid generic AI-looking design.

\- Keep current brand colors unless asked.

\- Do not break existing layout while fixing one section.



\## Communication

\- Explain changes in simple Bisaya/English.

\- Give exact commands when needed.

\- Be direct and do not overcomplicate.
